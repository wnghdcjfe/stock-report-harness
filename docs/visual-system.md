# Visual System — 최종 HTML 리포트 디자인 토큰

최종 산출물 `output/<slug>.html` 은 다음 디자인 시스템을 따른다.
`build-html.py` 는 이 토큰들을 그대로 CSS 변수로 출력한다.

## 색 (Light / Dark 동시 지원)

```
--bg-base:        #ffffff   |   #0a0a0a
--bg-surface:     #f7f7f8   |   #18181b
--bg-elevated:    #ffffff   |   #27272a
--text-strong:    #18181b   |   #fafafa
--text-base:      #3f3f46   |   #d4d4d8
--text-muted:     #71717a   |   #a1a1aa
--accent:         #2563eb   |   #60a5fa     (강조 / 차트 기본 색)
--up:             #16a34a   |   #4ade80     (주가 상승 / 긍정)
--down:           #dc2626   |   #f87171     (주가 하락 / 부정)
--border:         #e4e4e7   |   #3f3f46
```

**원칙**: 상승은 항상 `--up` (초록), 하락은 항상 `--down` (빨강). 한국 시장 관례와 동일. 차트 / 표 / 텍스트 강조 모두 통일.

## 타이포그래피

```
--font-sans: "Pretendard Variable", "Pretendard", -apple-system, sans-serif
--font-mono: "JetBrains Mono", ui-monospace, monospace

H1 (경제리포트 제목):     32px / 700 / -0.02em
H2 (섹션 제목):     24px / 700 / -0.01em
H3 (서브섹션):      18px / 600
본문:               16px / 400 / line-height 1.7
숫자/지표 강조:      monospace, tabular-nums
각주:               13px / 400 / --text-muted
```

## 레이아웃

- 리포트는 카드가 여러 장 쌓인 구조가 아니라 **하나의 통합 리포트 시트**로 렌더한다. `main.report-shell` 하나가 전체 본문을 감싸고, 개별 `<section>`에는 별도 카드 배경/라운드/그림자를 주지 않는다.
- 최대 본문 폭: 리포트 시트 기준 **792px**, 내부 텍스트 흐름은 약 **720px**. 가운데 정렬. 모바일에서는 시트 테두리 없이 18px 좌우 패딩.
- 섹션 간 구분은 카드가 아니라 **얇은 구분선과 제목 액센트 바**로 처리한다.
- 슬라이드 1개 = HTML `<section>` 1개. 각 섹션은 명시적 `id` (앵커 링크용).
- 좌측 또는 상단에 **목차 (TOC)** 자동 생성. sticky.
- 헤더 메타는 **날짜 1줄만 노출**. `난이도`, `약 N분`, `작성:` 같은 배지는 기본 숨김.

## 컴포넌트

### 통계 카드 (.stat-card)
숫자 위주 강조. 색이 부호를 표현 (`--up` / `--down`).
기본 카드형보다 **한 줄 강조형**을 우선한다.

```
영업이익 | 1Q26 연결 영업이익 | 57.2조원 | 1Q26 연결 매출 133.9조원
```

- 권장 구조:
  - label: `영업이익`
  - title: `1Q26 연결 영업이익`
  - value: `57.2조원`
  - compare: `1Q26 연결 매출 133.9조원`
- 줄바꿈보다 **가로 한 줄 강조**가 우선이며, 모바일에서만 2줄로 접힌다.
- 같은 섹션 안에서 통계 카드는 1~2개만 사용한다.

### 차트 (.chart)
- 라이브러리: **Chart.js v4**, CDN 임베드.
- 항상 `aria-label` 로 차트의 의미를 텍스트로도 제공 (스크린리더 / 검색 인덱싱).
- 시계열은 X축 날짜, Y축은 % 변화율을 기본으로 (절대값보다 의미 명확).

### 인용 블록 (.callout)
원문 인용 시. 좌측에 4px `--accent` border.

### References
- 본문 안에 각주 번호를 노출하지 않는다.
- 경제리포트 끝 References 섹션에만 매체-제목-날짜-URL 형식으로 모음.
- 숫자 라벨 없이 목록형 또는 불릿형으로 정리한다.

### 요약 목록 (.summary-list)
- 마지막 요약은 번호가 한 문단에 이어 붙지 않도록 한다.
- 다음처럼 **목록형 카드/리스트**로 한 줄씩 분리한다.

```
- 실적 발표 당일 반응과 한 달 추세는 다를 수 있다 — ...
- 최근 삼성전자의 핵심 동력은 DS와 HBM4였다 — ...
- 비용 부담 가능성도 함께 봐야 한다 — ...
```

## 다크모드

- `prefers-color-scheme: dark` 자동 감지.
- 우측 상단에 토글 버튼 (`☀ / ☾`).

## 인쇄/공유 친화

- 인쇄 시 차트 색 유지, 페이지 내 링크는 각주 번호로 표기되도록.
- OpenGraph 메타 태그 자동 삽입 (제목, 첫 섹션 요약, 대표 통계).
