# Pedagogy

리포트 구성 원칙입니다.

## 5-섹션 구조

1. **개요**: 왜 지금 봐야 하는지 한 문단으로 제시
2. **배경**: 기업/섹터/거시 맥락 정리
3. **메커니즘**: 왜 이런 일이 벌어졌는지 설명
4. **영향과 적용**: 다른 종목, 업황, 투자자 해석에 미치는 영향
5. **핵심 정리**: 30초 요약

## 가격 차트 원칙

- 가격 차트는 요청 기간 전체를 보여줘야 한다.
- 데이터는 반드시 `yfinance`의 실제 일별 데이터여야 한다.
- 일부 날짜만 발췌한 차트는 금지한다.
- draft에는 `price-chart` 블록만 선언하고, build가 데이터를 생성하게 한다.
- 1년 초과 요청은 가격 차트 없이 서술형 정리로 처리하거나 차단한다.

## 강조 원칙

- 핵심 결론은 `**굵게**` 사용 가능
- 리스트 다음 문단의 굵게도 유지
- 출처는 본문 footnote가 아니라 `References` 섹션에만 정리
