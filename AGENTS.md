## 2026-05-26 추가 운영 규칙

- 관련 종목(개별주/ETF/섹터 proxy)이 포함된 리서치는 해당 종목별로 **최신 뉴스 최소 100건**을 수집·분류·분석한다.
- `/research` 산출물에는 100건 뉴스의 날짜, 매체, 제목, 핵심 이슈, 가격/수급/리스크 해석을 표로 남긴다.
- `/draft`와 최종 HTML에는 100건 중 시간순 최신 5건을 별도 `최신 뉴스 5건 요약` 블록으로 노출한다. 각 항목은 날짜·매체·제목·1~2문장 요약·가격/수급 해석을 포함하고, 제목은 클릭 가능한 링크여야 한다.
- 뉴스 원자료 JSON에는 항목별 원문 URL(`url`, `article_url`, `google_news_link` 등)을 가능한 한 저장한다. 제공 API가 항목별 URL을 주지 않으면 원문 URL을 조작해 만들지 말고 제공사 뉴스 탭/검색 링크를 fallback으로 걸고 `url_is_fallback` 또는 본문 주석으로 명시한다.
- 한국 상장 종목은 가능하면 토스증권 종목 뉴스/투자자별 매매 동향을 참고하고, 사용한 API 또는 페이지 URL을 `sources`와 `output/assets/*` 원자료 JSON에 남긴다.
- 경제리포트 최종 HTML 본문에는 `[S1][S2]` 같은 인라인 참조 표식을 노출하지 않는다. 단, draft/research에는 검증용 출처 표식을 유지하고 References 섹션은 유지한다.

## Memory System 운영 규칙

반복 실패를 방지하기 위해 `docs/memory-system.md` 규칙을 따른다.

- 실패/재시도 비용이 큰 관측은 `memory/_daily/YYYY-MM-DD.md`에 append한다.
- 같은 패턴 3회 이상 또는 재발 비용이 큰 실패는 `memory/topics/{slug}.md`로 추출한다.
- topic 등록 후 같은 실패가 재발하면 텍스트 룰을 늘리지 말고 hook / linter / schema / test로 차단한다.
- memory 변경 후 `python3 scripts/validate_memory.py`를 실행한다.

작업 시작 시 해당 도메인의 topic만 읽는다.

| 도메인 | 파일 |
| --- | --- |
| 날짜 계산, 기간 계산, yfinance 기간, 거래일 라벨 | `memory/topics/time-sync.md` |
| OpenAI `image_gen`, 외부 뉴스/API 호출, yfinance 호출 | `memory/topics/external-api.md` |
| hero 이미지 후보/선택, 이미지 manifest, selected-image | `memory/topics/image-workflow.md` |
| `/plan`, `/research`, `/draft`, `/review`, `/build` 순서 | `memory/topics/pipeline-order.md` |
| `uv sync`, `pnpm install`, Python/Node 빌드 | `memory/topics/build-errors.md` |
| git commit / push / PR | `memory/topics/git-workflow.md` |
| Claude/Codex hook, validator, schema 차단 | `memory/topics/guardrails.md` |

자동 주입:
- 사용자 프롬프트 제출 시 `.claude/hooks/inject-memory-context.sh`가 `scripts/memory_context.py`를 실행해 관련 `memory/topics/*.md`를 컨텍스트에 주입한다.
- topic이 없거나 비어 있으면 작업을 막지 않고 상태만 알린다.
- 자동 주입은 읽기 보조이고, 실패 기록은 관측 발생 시 daily/topic 규칙에 따라 남긴다.
