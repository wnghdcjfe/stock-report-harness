# Stock Image Skill

Create the required hero image asset set for a report slug.
The image raster generation owner is a nested Codex CLI session using Codex's
`imagegen` skill. The outer Claude pipeline should orchestrate and verify the
file contract, not substitute local/procedural placeholder images.

## Preconditions

- `plan/<slug>.md`, `research/<slug>.md`, and `drafts/<slug>.md` must exist.
- Read all three files before writing prompts; title-only prompts are not acceptable.

## Procedure

1. Run the Codex CLI image executor:
   ```bash
   python3 scripts/run_stock_image_codex.py <slug>
   ```
   This wrapper:
   - reads `plan/<slug>.md`, `research/<slug>.md`, and `drafts/<slug>.md`
   - prepares the three prompt files
   - opens `codex exec` in this repository
   - instructs the nested Codex session to use `$imagegen` / built-in `image_gen`
   - verifies required PNG/JSON outputs after Codex exits
2. Required prompt outputs:
   - `output/assets/<slug>-hero-v1.prompt.txt`
   - `output/assets/<slug>-hero-v2.prompt.txt`
   - `output/assets/<slug>-hero-v3.prompt.txt`
3. Required real PNG candidates generated by Codex `imagegen`:
   - `output/assets/<slug>-hero-v1.png`
   - `output/assets/<slug>-hero-v2.png`
   - `output/assets/<slug>-hero-v3.png`
4. Required score JSON files:
   - `output/assets/<slug>-hero-v1.score.json` etc.
   - Include scores for report fit, company/sector fit, clarity, premium quality, and restraint.
5. Write:
   - `output/assets/<slug>-image-review.txt`
   - `output/assets/<slug>-image-manifest.json`
   - `output/assets/<slug>-selected-image.json`
   - Manifest JSON must include `status: complete`, `generation_method: codex-cli-imagegen`, and `generated_with`.
   - Selected-image JSON must include at least `slug`, `selected_candidate`, `image_path` or `selected_image`, `reason`, and `generated_with`.
6. Select exactly one hero image for final build.

For debugging only, inspect the command/prompt without generating images:
`python3 scripts/run_stock_image_codex.py <slug> --dry-run`.

## Image Rules

- No visible text, numbers, ticker symbols, logos, watermarks, UI screenshots, or fake dashboard screens.
- Use abstract/editorial visuals: wafers, memory stacks, market lines, macro/sector metaphors, studio lighting.
- The image must match the report’s actual tone; do not use generic bullish imagery for risk-heavy reports.
- Do not create procedural/Pillow/SVG/blank placeholder images as substitutes for Codex `imagegen`.
- Do not mark procedural/Pillow/SVG/placeholder output as `status: complete`.
- Do not call direct LLM/OpenAI API keys; use Codex CLI only.

## Blocking Rule

If Codex CLI or image generation is unavailable, the wrapper still writes the three prompt files and manifest with `status: blocked`, but you must not mark the image stage complete and must not allow `/stock-build` to pass.

## Completion Report

Report the selected image path, manifest path, and next command: `/stock-review <slug>`.

If invoked by `stock-goal`, this report is only an internal checkpoint. Do not stop or wait for the user; continue immediately to `/stock-review <slug>`.
